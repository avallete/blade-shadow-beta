Known issues

Especially on Linux, there are a lot of issues. This page describes well known issues, what is the cause and how to fix them when a workaround is available.

The application do not detect the trackpad

The Shadow application captures every input when the stream is on. However, the trackpad on some computer do not work at all except the mecanical click.

The Shadow application detects only mouse movement to control the mouse. Linux uses libinput, which detects the trackpad events as fingers action. This is visible inside the logs, when you tried to move the mouse with your trackpad. The logs will be spammed by a line like this: input error: Unhandled KEY 0x.... The codes are listed here.

Log line Event
input error: Unhandled KEY 0x14a (330) BTN_TOUCH
input error: Unhandled KEY 0x145 (325) BTN_TOOL_FINGER

There is no fix right know that will not disturb your system. We have to wait for a fix from Blade. You can try to remap the xinput events but it's a mess and may disturb the touchpad behaviour.

You can still use an external mouse and it will work perfectly. The unified client will propably fix this issue.

The application do not detect my gamepad

It's a well known issue, at the moment, Shadow on linux didn't detect any gamepad.

As a workaround, you can use Hamachi + Virtual Here to use your gamepad on shadow.

Virtual Here will allow you to capture the gamepad events, send them trought a network, and retranscribe them on another PC.

Hamachi will allow you to create the 'private network' for Virtual Here.

A tutorial is coming soon, giving detailled procedure.
